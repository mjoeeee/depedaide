Overview

Welcome to the Department of Education Portal for Assisting ICT Diagnosis and Enhancement System documentation. This guide will walk you through the features and functionalities available in the system.

Login

How to Log In

  1. Open your web browser and go to ozamiz.deped.gov.ph/depaide. You will be automatically redirected to the login page.
  2. Enter your DAWN username and password and click the Login button.
  3. If your credentials are correct, you will be redirected to the dashboard.

Role-Based Access

The DepAIDE System provides role-based access control to ensure users have the appropriate permissions. The available roles include:

Role Description
System Admin The admin can manage, process and set the status of the forms. They can as well edit, view and delete the forms
Personnel The personnel can fill up the certain request forms they want to be of service to. The status of their request forms are also visible.

Features

User Side

  • Dashboard: Upon logging in, users are greeted with a clean and organized dashboard that includes various service categories displayed as cards. A status of total requests is also shown on the cards of the available services. A calendar is presented as well, containing the requests set on certain dates. They can view the forms or services to be done on the specified dates by clicking it.
  • Request Status Calendar: This calendar displays all submitted requests by date, helping admins track activity and status trends throughout the month.
  • Printing ID Card: This request form is for the user who wishes to have a physical identity card (ID) to be printed. The personnel must first input the important details on the form and then click the “Update Details” button if they want to save the information they changed and redirect it to submitting a request for ID printing. A “Clear Field” button is also present if the user has misinput all the information in the input fields.
  • DepEd Email: On this form, the information needed for an email is linked to a DAWN account, which is why the system only requires the school ID, as this is the requirement to fulfill the creation of DepEd email accounts. The user will then click the “Apply Request” button to successfully submit the form.
  • Email Concern: This request form is for personnel who have concerns about their DepEd email accounts, such as forgotten passwords and other concerns regarding their email accounts. The personnel can submit a form raising the issues they are facing. The form includes an input box for the reason and a screenshot showing the email concern.
  • ICT Maintenance Request Form: This form is for the personnel within the division office who would like to request ICT Maintenance on their specific devices, property, and equipment that needs to be fixed or to inspect what the equipment is encountering and why it is working.
  • Documentation: This form is intended for personnel requesting documentation coverage for an event. The user must fill in the Event Title, Location of Event, Event Date and Time (including start and end times), and Details of the event. Once completed, the user clicks the "Apply Request" button to submit the request for documentation services.
  • Audio Visual Editing: This form is used for requesting audio-visual editing services. The personnel must provide the Project Title/Name, Project Description, and choose from dropdown options for Type of Project, Music Preferences, Expected Deliverables, Project Style & Tone, Preferred Delivery Method, and Project Deadline. After completing the required fields, the user submits the request by clicking the "Submit Form" button.
  • ICT Equipment Inspection: This form is for requesting an inspection of ICT equipment. It requires the Item/Description, Property Number, Property Acknowledgement Receipt No., Acquisition Cost, Acquisition Date, Complaints (if any), and the Nature and Scope of Last Repair. After entering all the necessary information, the user submits the form by clicking the "Submit Form" button.
  • Software Development: This form is for user or personnel who wishes to build a software program beneficial to both parties. Required information shall be filled in this form to further evaluate and assess the development of a program sysem or software.
  • Status: This section displays the current status of all submitted request forms. Each form type—such as ID Printing, DepEd Email, Email Concern, ICT Maintenance, Documentation, Audio Visual Editing, ICT Equipment Inspection, and Software Development—is shown along with its respective submission Date, Status (e.g., Pending, Approved, or Completed), and any Remarks provided by the ICT team.

Admin Side

  • Dashboard: The dashboard provides a quick summary of all request statuses across categories. Admins can view counts, monitor progress, and navigate to specific sections.
  • Request Status Calendar: This calendar displays all submitted requests by date, helping admins track activity and status trends throughout the month.
  • ID Card Requests: Shows all pending ID card applications with employee details. Admins can approve, reject, or view more information as needed.
  • Email Requests: This section tracks DepEd email requests. Admins can check reported concerns and provide responses, remarks or updates.
  • Email Concerns: This section tracks DepEd email issues like login problems. Admins can check reported concerns and provide responses or updates.
  • Audio Visual Editing: This page shows all AV project requests, including deadlines and progress. Admins can update project info and monitor completion status.
  • Documentation Data: Displays documentation tasks and events for tracking purposes. Admins can view task details and update their statuses.
  • ICT Maintenance Requests: Lists maintenance requests related to ICT concerns. Admins can review, approve, or reject tasks based on the issue.
  • ICT Equipment Inspection: Lists equipment inspection requests with property details and complaints. Admins can edit records or add remarks and use the search tool.
  • Software Development Requests: Displays all software/system development requests with descriptions and deadlines. Admins can edit details, add remarks, and search entries.
